## 一、研究背景与动机

### 问题陈述

```
┌─────────────────────────────────────────────────────────────────┐
│                    LVLM 量化困境                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  大型视觉 - 语言模型 (LLaVA 等) 部署困难                         │
│  ├─ 模型太大 (13B+ 参数)                                        │
│  ├─ 推理慢 (多模态 + 自回归生成)                                │
│  └─ 无法在移动端/机器人部署                                     │
│                                                                 │
│  现有 PTQ 方法 (GPTQ/AWQ) 的问题:                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  逐层贪心搜索 → 忽略跨层依赖 → 误差累积 → 精度大幅下降 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 核心洞察

> **关键发现:** 激活熵 (Activation Entropy) 与 跨层依赖强度 强相关

```
高熵层 → 激活分布复杂 → 量化噪声大 → 对后续层影响大
低熵层 → 激活分布集中 → 量化鲁棒 → 影响小

相关系数达 0.82 (论文 Figure 2)
```

## 二、方法详解

### 2.1 整体框架

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Q-VLM 三阶段流程                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐          │
│  │  Stage 1     │    │  Stage 2     │    │  Stage 3     │          │
│  │  熵分析      │ →  │  Block 划分   │ →  │  联合搜索    │          │
│  └──────────────┘    └──────────────┘    └──────────────┘          │
│       │                   │                   │                     │
│       ▼                   ▼                   ▼                     │
│  用 calibration data   基于熵值相似性      Block 内逐层搜索        │
│  前向传播收集激活      划分连续层为 block    rounding offset        │
│  计算每层熵值          高熵层单独处理        最小化 block 输出误差   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 核心概念定义

#### 激活熵 (Activation Entropy)

```
计算对象：每层的输出激活 (不是权重!)

计算方法:
  1. 用 128-512 个 calibration sample 前向传播
  2. 收集某层的激活值 (shape: [batch, seq, dim])
  3. 展平为一维数组
  4. 计算直方图 (256 bins)
  5. 计算香农熵: H = -Σ p(i) × log₂(p(i))

物理含义:
  ┌────────────────────────────────────────────────────┐
  │  低熵 (H≈2-4 bits)  → 激活集中 → 量化不敏感       │
  │  高熵 (H≈6-8 bits)  → 激活分散 → 量化敏感         │
  └────────────────────────────────────────────────────┘
```

#### Rounding Offset

```
定义：量化取整时的偏移参数

量化公式 (分步版):
  ┌─────────────────────────────────────────────────────┐
  │  Step 1: 归一化    x' = (w - z) / s                 │
  │  Step 2: 加 offset   x'' = x' + δ      ← 搜索这个!  │
  │  Step 3: 取整      q = round(x'')                   │
  │  Step 4: 反量化    ŵ = q × s + z                   │
  └─────────────────────────────────────────────────────┘

参数说明:
  w     : 原始浮点权重
  s     : scale (预先计算，基于权重范围)
  z     : zero-point (对称量化时为 0)
  δ     : rounding offset ∈ [-0.5, 0.5]  ← 搜索变量
  q     : 量化后整数
  ŵ     : 量化后权重 (反量化回浮点表示)

搜索空间:
  范围：[-0.5, 0.5]  (固定)
  候选：21 个点 (步长 0.05) 或 41 个点 (步长 0.025)
  粒度：Per-Layer (每个 linear layer 一个独立 offset)
```

#### Block 划分

```
原则:
  ┌──────────────────────────────────────────────────────────┐
  │  1. 熵值相近的层 → 合并到同一 block                       │
  │  2. 高熵层 (熵>阈值) → 单独成 block                      │
  │  3. 关键层边界 (Attention 输出等) → 分割点               │
  │  4. block 大小上限：4-6 层 (防止搜索空间爆炸)             │
  └──────────────────────────────────────────────────────────┘

示例 (LLaVA-13B, 40 层):
  
  层号    熵值    Block 分配
  ─────────────────────────
  0-3     6.1-6.4   Block 1 (低熵合并)
  4-7     7.0-7.3   Block 2 (中熵合并)
  8       8.9       Block 3 (高熵单独!)
  9-14    6.5-6.8   Block 4 (低熵合并)
  15      9.1       Block 5 (高熵单独!)
  16-39   5.8-6.5   Block 6-10 (稳定大 block)
```

### 2.3 Block-wise 搜索算法

```
┌─────────────────────────────────────────────────────────────────┐
│              Block 内联合搜索流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  假设 Block 包含 3 层 (L0, L1, L2)                               │
│                                                                 │
│  【迭代搜索】                                                   │
│                                                                 │
│  Iteration 1: 搜索 L0 的 δ₀                                     │
│    候选 δ₀: -0.50, -0.45, ..., 0, ..., +0.45, +0.50 (21 个)    │
│    对每个候选:                                                  │
│      1. 用δ₀量化 L0                                             │
│      2. L1, L2 保持浮点                                         │
│      3. Forward: 输入 → [L0(量化)] → [L1(浮点)] → [L2(浮点)] → 输出 │
│      4. 计算: ||输出_quant - 输出_fp||²                         │
│    选误差最小的 δ₀*                                              │
│                                                                 │
│  Iteration 2: 搜索 L1 的 δ₁ (固定δ₀*)                           │
│    对每个候选δ₁:                                                │
│      1. L0 用δ₀* (已固定)                                       │
│      2. 用δ₁量化 L1                                             │
│      3. L2 保持浮点                                             │
│      4. Forward 整个 block，计算输出误差                        │
│    选误差最小的 δ₁*                                              │
│                                                                 │
│  Iteration 3: 搜索 L2 的 δ₂ (固定δ₀*, δ₁*)                      │
│    ... 同理 ...                                                 │
│                                                                 │
│  输出：Block 内最优 offset 组合 (δ₀*, δ₁*, δ₂*)                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
