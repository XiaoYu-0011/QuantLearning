# ARCQuant 核心步骤图解

---

## 步骤 1：找出刺头通道

假设激活矩阵 X 有 8 个通道（实际可能有几千个），每个通道的最大幅值如下：

```
通道编号:    ch0   ch1   ch2   ch3   ch4   ch5   ch6   ch7
最大幅值:    0.5   85    1.2   0.8   92    0.3   1.1   0.7
                   ^^^              ^^^
                  刺头！            刺头！
```

**按幅值排序（大的放前面）：**

```
排序后:      ch4   ch1   ch2   ch6   ch3   ch7   ch0   ch5
最大幅值:    92    85    1.2   1.1   0.8   0.7   0.5   0.3
             ├─ 刺头区 ─┤     ├──────── 正常区 ────────┤
                S = 2 个                  K-S = 6 个
```

**阈值怎么定？**

```
层最大值 M = 92

阈值 τ = 2⁻³ × M = 92 / 8 = 11.5

超过 11.5 的通道 → 刺头（ch4=92, ch1=85）
低于 11.5 的通道 → 正常（NVFP4 精度已够用）
```

> 为什么是 2⁻³？因为 NVFP4 的指数是 E2M1（2-bit），参考格式 FP8 的指数是 E5M2（5-bit），
> 差了 3 bit，所以阈值是 2⁻³ 倍的最大值。低于这个值的数，NVFP4 和 FP8 精度差不多。

## 步骤 2：量化两遍（以刺头通道 ch4 为例）

### 第一遍：正常 NVFP4 量化

```
原始值 Xo:          [92.0,  87.3,  -91.5,  78.2, ...]
                         │
                         ▼
               ┌─────────────────────┐
               │  NVFP4 量化 (E2M1)  │
               │                     │
               │  缩放因子 s₁ = 32   │
               │  Q_Xo = round(X/s₁) │
               └─────────────────────┘
                         │
                         ▼
量化整数 Q_Xo:      [3,     3,     -3,     2, ...]

还原值 s₁·Q_Xo:    [96.0,  96.0,  -96.0,  64.0, ...]
```

### 算残差：原始值 - 还原值

```
原始值:              92.0    87.3   -91.5    78.2
还原值:              96.0    96.0   -96.0    64.0
                     ────    ────    ────    ────
残差 Ro:             -4.0    -8.7     4.5    14.2
                      ↑
              这就是第一遍量化丢掉的信息！
```

**公式：**  $R_o = X_o - s_1 \cdot Q_{X_o}$

### 第二遍：对残差再做一次 NVFP4 量化

```
残差 Ro:             [-4.0,  -8.7,   4.5,   14.2, ...]
                         │
                         ▼
               ┌─────────────────────┐
               │  NVFP4 量化 (E2M1)  │
               │                     │
               │  缩放因子 s₂ = 6    │
               │  Q_Ro = round(R/s₂) │
               └─────────────────────┘
                         │
                         ▼
量化整数 Q_Ro:       [-1,    -1,      1,     2, ...]

还原值 s₂·Q_Ro:     [-6.0,  -6.0,   6.0,   12.0, ...]
```

### 两遍加起来 vs 只做一遍

```
                        ch4 的某个值: 原始 = 87.3

  只做一遍:     s₁·Q_Xo        = 96.0          误差 = 8.7
  做两遍:       s₁·Q_Xo + s₂·Q_Ro = 96.0 + (-6.0) = 90.0    误差 = 2.7
                                                                ↓
                                                          误差缩小了 3 倍！
```

> 大白话：第一遍量化把 87.3 记成了 96.0（差了 8.7），
> 第二遍专门量化这个 -8.7 的误差，修正回 90.0（只差 2.7 了）。

## 步骤 3：拼接进矩阵乘法

这是最精妙的部分。我们看看怎么把"两遍量化"变成"一次矩阵乘法"。

### 原始计算（没有量化时）

```
Y = X × W⊤

X: (N × K)       W: (M × K)

    ┌─────────────────┐       ┌─────────────────┐⊤
    │   K=8 个通道     │       │   K=8 个通道     │
    │                 │   ×   │                 │   =  Y (N × M)
    │  N 个 token     │       │  M 个输出通道    │
    └─────────────────┘       └─────────────────┘
```

### ARCQuant 的拼接操作

**激活侧：**

```
原始 X（排序后）:
┌──────────────────────────────────────┐
│  ch4  ch1 │ ch2  ch6  ch3  ch7  ch0  ch5 │
│  刺头(S=2) │        正常通道(K-S=6)        │
└──────────────────────────────────────┘
      │                    │
      ▼                    ▼
   ┌──────┐          ┌─────────────────────┐
   │NVFP4 │          │      NVFP4 量化      │
   │量化   │          │                     │
   └──┬───┘          └──────────┬──────────┘
      │                         │
      ▼                         │
  算残差 Ro                      │
      │                         │
      ▼                         │
   ┌──────┐                     │
   │NVFP4 │                     │
   │量化   │                     │
   └──┬───┘                     │
      │                         │
      ▼                         ▼

Q_X_aug（增强后的激活矩阵）:
┌──────────────────────────────────────────────────┐
│      Q_X（全部 K 个通道的主量化）     │ Q_Ro（S个残差）│
│  Q_ch4  Q_ch1  Q_ch2 ... Q_ch5       │ Ro_ch4  Ro_ch1│
│◄──────────── K = 8 ─────────────────►│◄── S = 2 ──►│
└──────────────────────────────────────────────────┘
                     总宽度 = K + S = 10
```

**权重侧（离线准备好）：**

```
Q_W_aug（增强后的权重矩阵）:
┌──────────────────────────────────────────────────┐
│     Q_W（全部 K 个通道的量化权重）     │ Q_Wo（复制的）│
│  Q_ch4  Q_ch1  Q_ch2 ... Q_ch5        │ Q_ch4  Q_ch1 │
│◄──────────── K = 8 ──────────────────►│◄── S = 2 ──►│
└──────────────────────────────────────────────────┘
                     总宽度 = K + S = 10
```

> 权重侧更简单：刺头通道的权重直接复制一份贴在后面。

### 矩阵乘法自动完成修正！

```
Y = Q_X_aug × Q_W_aug⊤

  ┌──────────┬────────┐     ┌──────────┬────────┐⊤
  │          │        │     │          │        │
  │  Q_X     │  Q_Ro  │  ×  │  Q_W     │  Q_Wo  │
  │ (N×K)    │ (N×S)  │     │ (M×K)    │ (M×S)  │
  │          │        │     │          │        │
  └──────────┴────────┘     └──────────┴────────┘

  展开后等价于：

  Y = Q_X · Q_W⊤   +   Q_Ro · Q_Wo⊤
      ─────────────     ──────────────
      主量化结果          残差修正项
      (正常的4bit计算)    (自动补偿误差)
```

**公式：**

$$Y \approx \underbrace{Q(X) \cdot Q(W)^\top}_{\text{主量化（所有通道）}} + \underbrace{Q(R_o) \cdot Q(W_o)^\top}_{\text{残差修正（刺头通道）}}$$

$$= s_{X_{aug}} \cdot Q_{X_{aug}} \cdot (s_{W_{aug}} \cdot Q_{W_{aug}})^\top$$

### 为什么这样做是对的？

矩阵乘法本质上是"逐列相乘再求和"：

```
Y[i][j] = Σ(k=0 to K+S-1) X_aug[i][k] × W_aug[j][k]

        = Σ(k=0 to K-1) Q_X[i][k] × Q_W[j][k]     ← 前 K 列：主量化
        + Σ(k=K to K+S-1) Q_Ro[i][k-K] × Q_Wo[j][k-K]  ← 后 S 列：残差修正
          ─────────────────────────────────────────────
                        自动求和，不需要任何额外操作！
```

> 大白话：矩阵乘法本来就是把每一列的贡献加起来。
> 我在后面多拼了 S 列"修正值"，矩阵乘法自然就把修正值也加上了。
> 不需要改任何硬件指令，不需要写特殊的计算逻辑。
> GPU 看到的就是一个 (K+S) 宽的标准 NVFP4 矩阵乘法。

## 完整流程图

```
                        ┌─────────────────┐
                        │  校准数据（离线）  │
                        └────────┬────────┘
                                 │
                    ┌────────────┴────────────┐
                    ▼                         ▼
           ┌───────────────┐        ┌────────────────┐
           │ 统计通道幅值    │        │  确定阈值 τ     │
           │ 按大小排序     │        │  = 2⁻³ × M     │
           └───────┬───────┘        └────────┬───────┘
                   │                         │
                   └────────────┬─────────────┘
                                │
                   ┌────────────┴────────────┐
                   │  标记刺头通道 (top-S个)   │
                   │  记录排序索引            │  ← 步骤 1
                   └────────────┬────────────┘
                                │
              ┌─────────────────┴─────────────────┐
              │                                   │
              ▼                                   ▼
    ┌──────────────────┐                ┌──────────────────┐
    │  权重 W（离线）    │                │  激活 X（在线）    │
    │                  │                │                  │
    │ 1. 按索引重排序    │                │ 1. 按索引重排序    │
    │ 2. NVFP4 量化→Q_W │                │ 2. NVFP4 量化→Q_X │
    │ 3. 复制刺头权重    │                │ 3. 刺头算残差 Ro   │  ← 步骤 2
    │    Q_Wo = Q_W[刺头]│                │ 4. NVFP4量化→Q_Ro │
    │ 4. 拼接:          │                │ 5. 拼接:          │
    │    [Q_W | Q_Wo]   │                │    [Q_X | Q_Ro]   │  ← 步骤 3
    └────────┬─────────┘                └────────┬─────────┘
             │                                   │
             │         全部都是 NVFP4 格式！        │
             │                                   │
             └─────────────┬─────────────────────┘
                           │
                           ▼
                 ┌──────────────────┐
                 │  标准 NVFP4 GEMM  │
                 │  (K+S) 宽度      │
                 │  硬件完全无感！    │
                 └────────┬─────────┘
                          │
                          ▼
                    输出 Y ≈ XW⊤
               （精度接近 W4A8！速度接近 W4A4！）
```

## 数值例子完整走一遍

假设 K=4, S=1（只有 1 个刺头通道），1 个 token，2 个输出：

```
原始激活 X = [80.0, 1.5, 0.8, 1.2]     ← ch0=80 是刺头
原始权重 W = [[2.0, 0.5, 1.0, 0.3],    ← 输出通道 0
              [1.0, 0.8, 0.5, 1.0]]    ← 输出通道 1

真实结果 Y = X × W⊤
Y[0] = 80×2.0 + 1.5×0.5 + 0.8×1.0 + 1.2×0.3 = 161.91
Y[1] = 80×1.0 + 1.5×0.8 + 0.8×0.5 + 1.2×1.0 = 82.80
```

**不用 ARCQuant（直接 NVFP4 量化）：**

```
Q(X) ≈ [80, 1.5, 1.0, 1.0]    （刺头 ch0 的量化还好）
但如果 80 被量化成 64（NVFP4 只有 4bit）：
Q(X) ≈ [64, 1.5, 1.0, 1.0]

Y_naive[0] = 64×2.0 + 1.5×0.5 + 1.0×1.0 + 1.0×0.3 = 130.05
真实值 = 161.91，误差 = 31.86  ← 很大！
```

**用 ARCQuant：**

```
第一遍量化：Q_X = [64, 1.5, 1.0, 1.0]（和上面一样）

刺头 ch0 的残差：Ro = 80 - 64 = 16
第二遍量化残差：Q_Ro = [16]（假设能精确表示）

拼接后：
Q_X_aug = [64,  1.5, 1.0, 1.0, | 16  ]    ← K+S = 5 列
Q_W_aug = [[2.0, 0.5, 1.0, 0.3, | 2.0 ],   ← 复制 ch0 的权重
            [1.0, 0.8, 0.5, 1.0, | 1.0 ]]

Y_arc[0] = 64×2.0 + 1.5×0.5 + 1.0×1.0 + 1.0×0.3 + 16×2.0
         = 128 + 0.75 + 1.0 + 0.3 + 32
         = 162.05
真实值 = 161.91，误差 = 0.14  ← 几乎完美！

对比：直接量化误差 31.86 → ARCQuant 误差 0.14，缩小了 200 多倍！
```
