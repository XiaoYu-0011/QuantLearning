## 二、① 离线校准：找出刺头通道（Adaptive Outlier Identification）

```
输入：校准数据集（128条文本）
      ┃
      ▼
┌─────────────────────────────────────────────────────────────┐
│  用校准数据跑一遍模型，统计每层每个通道的激活最大幅值          │
└─────────────────────────────┬───────────────────────────────┘
                              ▼
         通道:  ch0   ch1   ch2   ch3   ch4   ch5   ch6   ch7
         幅值:  0.5   85    1.2   0.8   92    0.3   1.1   0.7
                              │
                              ▼
                  ┌───────────────────────┐
                  │ 按幅值从大到小排序      │
                  └───────────┬───────────┘
                              ▼
         排序后: ch4   ch1   ch2   ch6   ch3   ch7   ch0   ch5
         幅值:   92    85    1.2   1.1   0.8   0.7   0.5   0.3
                              │
                              ▼
              ┌────────────────────────────────┐
              │ 计算阈值                        │
              │                                │
              │ M = max(所有通道) = 92          │
              │ τ = 2⁻³ × M = 92 / 8 = 11.5   │
              │                                │
              │ 为什么是 2⁻³？                  │
              │ FP8指数5bit - NVFP4指数2bit = 3 │
              └────────────────┬───────────────┘
                               ▼
              ┌────────────────────────────────┐
              │ 超过阈值 → 标记为刺头            │
              │                                │
              │ ch4=92  > 11.5 → 刺头 ✅        │
              │ ch1=85  > 11.5 → 刺头 ✅        │
              │ ch2=1.2 < 11.5 → 正常 ❌        │
              │ ...其余都 < 11.5 → 正常 ❌       │
              └────────────────┬───────────────┘
                               ▼
              ┌────────────────────────────────┐
              │ 输出（保存下来，推理时直接用）：   │
              │                                │
              │  • 通道排序索引                  │
              │  • 刺头通道数量 S = 2            │
              └────────────────────────────────┘
```

## 三、② 离线权重量化（Offline Weight Quantization）

```
输入：原始权重 W (M × K)，排序索引，S
      ┃
      ▼
┌──────────────────────────────────────────────────────────────────┐
│ Step 1: 按排序索引重排列权重的列                                   │
│                                                                  │
│  原始 W:     [w_ch0, w_ch1, w_ch2, w_ch3, w_ch4, w_ch5, ...]    │
│                               ↓ 重排序                           │
│  W_sorted:   [w_ch4, w_ch1, w_ch2, w_ch6, w_ch3, w_ch7, ...]    │
│               刺头↑    刺头↑   ←── 正常通道 ──→                    │
└──────────────────────────────────┬───────────────────────────────┘
                                   ▼
┌──────────────────────────────────────────────────────────────────┐
│ Step 2: 对全部通道做 NVFP4 量化                                    │
│                                                                  │
│  Q_W = NVFP4_Quantize(W_sorted)                                  │
│                                                                  │
│  Q_W:  [Q_ch4, Q_ch1, Q_ch2, Q_ch6, Q_ch3, Q_ch7, Q_ch0, Q_ch5]│
│         ◄── K = 8 个通道 ──────────────────────────────────────► │
└──────────────────────────────────┬───────────────────────────────┘
                                   ▼
┌──────────────────────────────────────────────────────────────────┐
│ Step 3: 复制刺头通道的权重，拼到后面                                │
│                                                                  │
│  Q_Wo = Q_W 中刺头通道的副本 = [Q_ch4, Q_ch1]                     │
│                                                                  │
│  Q_W_aug = [ Q_W (K=8列)              |  Q_Wo (S=2列)  ]        │
│          = [Q_ch4 Q_ch1 ... Q_ch5     |  Q_ch4  Q_ch1  ]        │
│            ◄──── 原始权重量化 ────────►  ◄─ 复制刺头 ──►          │
│            ◄──────────── 总宽度 K+S = 10 ───────────────►         │
│                                                                  │
│  ⚠️ 注意：不需要计算残差！直接复制就行！                             │
│     因为残差在激活侧算，权重侧只需要匹配维度                        │
└──────────────────────────────────┬───────────────────────────────┘
                                   ▼
                        ┌────────────────────┐
                        │ 保存 Q_W_aug 到磁盘  │
                        │ 推理时直接加载       │
                        └────────────────────┘
```

## 四、③ 在线激活量化（Online Activation Quantization）

**这是推理时每个 token 都要做的，所以要快！**

```
输入：当前激活 X (N × K)，排序索引，S
      ┃
      ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: 重排序 + 主量化                                              │
│                                                                     │
│  原始 X:    [x_ch0, x_ch1, x_ch2, x_ch3, x_ch4, x_ch5, ...]       │
│                              ↓ 按校准时的索引重排序                   │
│  X_sorted:  [x_ch4, x_ch1, x_ch2, x_ch6, x_ch3, x_ch7, ...]       │
│              刺头↑    刺头↑   ←────── 正常通道 ──────→                │
│                              ↓ NVFP4 块级量化                        │
│  Q_X:       [Q_ch4, Q_ch1, Q_ch2, Q_ch6, ...]                      │
│                                                                     │
│  同时记录刺头通道的缩放因子 s_Xo                                      │
└─────────────────────────────────┬───────────────────────────────────┘
                                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 2: 残差补偿（只对 S 个刺头通道做！）                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────┐      │
│  │ 以 ch4 为例（原始值 = 92.0，量化还原值 = 96.0）：           │      │
│  │                                                           │      │
│  │   原始值 Xo     92.0                                      │      │
│  │                  │                                        │      │
│  │                  ▼                                        │      │
│  │            ┌──────────┐                                   │      │
│  │            │ NVFP4量化 │ ──→ Q_Xo = 3, s₁ = 32           │      │
│  │            └──────────┘                                   │      │
│  │                  │                                        │      │
│  │                  ▼                                        │      │
│  │            还原值 = s₁ × Q_Xo = 32 × 3 = 96.0            │      │
│  │                  │                                        │      │
│  │                  ▼                                        │      │
│  │    ┌─────────────────────────────┐                        │      │
│  │    │ 残差 Ro = 原始 - 还原         │                        │      │
│  │    │       = 92.0 - 96.0 = -4.0  │                        │      │
│  │    └──────────────┬──────────────┘                        │      │
│  │                   ▼                                       │      │
│  │             ┌──────────┐                                  │      │
│  │             │ NVFP4量化 │ ──→ Q_Ro = -1, s₂ = 6          │      │
│  │             └──────────┘                                  │      │
│  │                   │                                       │      │
│  │                   ▼                                       │      │
│  │   两遍加起来 = 96.0 + (-6.0) = 90.0  ≈ 92.0 ✅            │      │
│  │   （只做一遍 = 96.0，误差 4.0 → 做两遍误差仅 2.0）          │      │
│  └───────────────────────────────────────────────────────────┘      │
│                                                                     │
│  公式：  Ro = Xo − s_Xo · Q_Xo                                      │
│         Q_Ro = NVFP4_Quantize(Ro)                                   │
└─────────────────────────────────┬───────────────────────────────────┘
                                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 3: 拼接增强                                                     │
│                                                                     │
│  Q_X（全部 K 个通道的主量化）：                                        │
│  ┌──────────────────────────────────────┐                           │
│  │ Q_ch4  Q_ch1  Q_ch2  Q_ch6 ... Q_ch5│                           │
│  │◄────────────── K = 8 ──────────────►│                           │
│  └──────────────────────────────────────┘                           │
│                                                                     │
│  Q_Ro（S 个刺头通道的残差量化）：                                      │
│  ┌──────────┐                                                       │
│  │Ro_ch4 Ro_ch1│                                                    │
│  │◄── S=2 ──►│                                                     │
│  └──────────┘                                                       │
│                                                                     │
│  拼接：                                                              │
│  Q_X_aug = [Q_X | Q_Ro]                                            │
│  ┌──────────────────────────────────────┬───────────┐               │
│  │ Q_ch4  Q_ch1  Q_ch2  Q_ch6 ... Q_ch5│Ro_ch4 Ro_ch1│              │
│  │◄────────── 主量化 K=8 ──────────────►│◄─残差 S=2─►│              │
│  └──────────────────────────────────────┴───────────┘               │
│  ◄──────────────── 总宽度 K+S = 10 ────────────────►                │
│                                                                     │
│  scales 也拼接：s_X_aug = [s_X | s_Ro]                               │
│                                                                     │
│  ⚠️ 全部都是 NVFP4 格式！没有混合精度！                                │
└─────────────────────────────────────────────────────────────────────┘
```

## 五、④ 统一 GEMM 执行

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Q_X_aug (N × K+S)                 Q_W_aug (M × K+S)                │
│  ┌──────────────┬──────┐           ┌──────────────┬──────┐          │
│  │              │      │           │              │      │          │
│  │   Q_X        │ Q_Ro │           │   Q_W        │ Q_Wo │          │
│  │  主量化       │ 残差  │     ×     │  权重量化     │ 复制  │⊤         │
│  │  (N × K)     │(N×S) │           │  (M × K)     │(M×S) │          │
│  │              │      │           │              │      │          │
│  └──────────────┴──────┘           └──────────────┴──────┘          │
│         全是 NVFP4                        全是 NVFP4                  │
│                                                                      │
│                          ↓ 展开                                       │
│                                                                      │
│  Y = Q_X_aug × Q_W_aug⊤                                             │
│                                                                      │
│    = [Q_X | Q_Ro] × [Q_W | Q_Wo]⊤                                   │
│                                                                      │
│    = Q_X · Q_W⊤     +     Q_Ro · Q_Wo⊤                              │
│      ───────────           ─────────────                             │
│      主量化结果              残差修正项                                 │
│      (所有通道的             (刺头通道的                                │
│       粗略计算)               精度补偿)                                │
│                                                                      │
│                          ↓                                            │
│                                                                      │
│  ┌───────────────────────────────────────────────────┐               │
│  │          标准 NVFP4 GEMM 内核                      │               │
│  │                                                   │               │
│  │  • 硬件看到的就是一个 (K+S) 宽的统一 NVFP4 矩阵乘法 │               │
│  │  • 直接用 CUTLASS 优化内核，不需要任何修改           │               │
│  │  • Tensor Core 全速运行                            │               │
│  │  • S 通常很小 (几十~几百)，额外开销 < 5%            │               │
│  └───────────────────────────────────────────────────┘               │
│                          ↓                                            │
│                    输出 Y (N × M)                                     │
│               精度 ≈ W4A8，速度 ≈ W4A4                                │
└──────────────────────────────────────────────────────────────────────┘
```

## 六、融合内核：为什么在线部分不会太慢？

**问题：在线激活量化要做排序、RMSNorm、主量化、算残差、残差量化，这么多步不慢吗？**

```
  普通做法（5 次内存读写）：            ARCQuant 融合内核（1 次内存读写）：

  ┌────────────┐                      ┌──────────────────────────────┐
  │ RMSNorm    │ ←读→ 显存             │                              │
  └─────┬──────┘                      │  一个 CUDA Kernel 里全做完：   │
        ▼                             │                              │
  ┌────────────┐                      │  ① RMSNorm                   │
  │ 通道重排序  │ ←读→ 显存             │        ↓                     │
  └─────┬──────┘                      │  ② 通道重排序                 │
        ▼                             │        ↓                     │
  ┌────────────┐                      │  ③ 主量化 → Q_X              │  ←读→ 显存
  │ 主量化      │ ←读→ 显存             │        ↓                     │  （只读写
  └─────┬──────┘                      │  ④ 算残差 Ro                  │    一次！）
        ▼                             │        ↓                     │
  ┌────────────┐                      │  ⑤ 残差量化 → Q_Ro            │
  │ 算残差      │ ←读→ 显存             │        ↓                     │
  └─────┬──────┘                      │  ⑥ 输出 Q_X_aug              │
        ▼                             │                              │
  ┌────────────┐                      └──────────────────────────────┘
  │ 残差量化    │ ←读→ 显存
  └────────────┘                      GPU 计算很快，瓶颈在内存读写
                                      5次 → 1次，延迟大幅降低
  慢！来回搬数据 5 次                    快！数据在寄存器里流转
```

## 七、讲解用的"一页纸总结"

```
╔══════════════════════════════════════════════════════════════════╗
║                ARCQuant 核心思路（一页纸版）                      ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  问题：LLM 激活里少数通道值特别大（刺头），4-bit 量化误差爆炸      ║
║                                                                  ║
║  别人的方案          为什么不行                                    ║
║  ──────────          ──────────                                   ║
║  旋转打散             破坏 NVFP4 块隔离，越旋越差                  ║
║  混合精度             硬件不支持混合块大小，跑不快                  ║
║  平滑量化             4-bit 下权重扛不住                           ║
║                                                                  ║
║  ─────────────────────────────────────────────────────            ║
║                                                                  ║
║  ARCQuant 的方案：对刺头通道"补一刀"                              ║
║                                                                  ║
║  ① 找刺头    校准数据统计 → 按幅值排序 → 阈值 τ=M/8 筛选          ║
║                                                                  ║
║  ② 量化两遍  第一遍正常量化 → 算残差 → 第二遍量化残差              ║
║                                                                  ║
║  ③ 拼进GEMM  激活: [Q_X | Q_Ro]   权重: [Q_W | Q_Wo]             ║
║              矩阵乘法自动把修正项加上，硬件完全无感                 ║
║                                                                  ║
║  ─────────────────────────────────────────────────────            ║
║                                                                  ║
║  结果：精度 ≈ W4A8    速度 ≈ W4A4    额外开销 < 5%                ║
║                                                                  ║
║  理论保证：两遍4bit误差 = (2⁻²)² = 2⁻⁴ = 一遍8bit误差             ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝
```
