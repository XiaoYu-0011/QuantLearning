🔬 SAQ-SAM

语义对齐量化 for Segment Anything Model

AAAI 2026 | arXiv:2503.06515

❓ Motivation - 为什么要做这个？

🎯 核心问题

Segment Anything Model (SAM) 性能强大，但模型太大、计算太贵，难以在边缘设备部署。后训练量化 (PTQ) 是理想的压缩方案，但直接套用现有 PTQ 方法效果很差。

⚠️ 三大挑战

  • 极端激活异常值 — SAM 的注意力机制中存在极端的激活异常值，传统量化器难以处理
  • 语义信息丢失 — 传统 PTQ 只关注"数值误差最小"，忽略了"语义一致性"
  • 跨模态交互受损 — SAM 的核心是图像 - 提示交互，量化破坏了这种交互的语义对齐

❌ 传统 PTQ 思路

最小化数值重建误差 (MSE)
→ 量化后数值接近
→ 但分割性能大幅下降

✅ SAQ-SAM 思路

最小化语义一致性损失
→ 量化后注意力模式一致
→ 分割性能有效保持

💡 Insight - 关键洞察是什么？

作者通过深入分析发现了几个反直觉的现象：

🎯 洞察 1：语义对齐 > 数值精度

即使量化数值误差较大，只要注意力焦点区域一致，分割性能就能保持。数值精度 ≠ 语义能力。

👁️ 洞察 2：注意力 IoU 是好的代理指标

量化前后注意力图的 IoU（重叠度）与最终分割性能高度相关，可作为量化优化的目标。

🔄 洞察 3：图像 - 提示交互需要双向保护

Mask Decoder 中的 Cross-Attention 交互模式需要同时考虑分布对齐和语义对齐。

📈 洞察 4：异常值是信号不是噪声

极端激活异常值通常对应重要语义信息，应用来指导裁剪而非抑制。

🛠️ Solution - 怎么解决的？

1️⃣ PCC - 感知一致性裁剪 针对激活异常值

核心思想： 用注意力 IoU 损失指导裁剪范围搜索，而不是传统的基于统计的方法。

步骤：
1. 获取 FP 模型的注意力图 ATM_FP
2. 对不同的裁剪百分比 pct ∈ [0.85, 0.9999]
3. 计算量化注意力图 ATM_Quant(pct)
4. 选择使 IoU(ATM_FP, ATM_Quant) 最大的 pct

损失函数： AttentionOverlapLoss = 1 - IoU(高注意力区域)

效果： 仅 PCC 就能在 W6A6 下带来显著提升，对 SAM-B 效果最明显。

2️⃣ PAR - 提示感知重建 针对跨模态交互

核心思想： 在 Encoder 量化重建时，不仅重建 Encoder 输出，还要重建 Mask Decoder 中的 Cross-Attention 交互响应。

重建流程：
1. 量化 Encoder → 输出 Image Token
2. Image Token + Prompt → 通过 Mask Decoder Cross-Attention
3. 获取交互后的 Cross Image Token (CIT)
4. 计算损失：L = β·L_rec + α·L_PAR
- L_rec: Encoder 输出重建 MSE
- L_PAR: CIT 的交互相似性损失

关键： L_PAR 保护了图像 - 提示交互的语义一致性

3️⃣ 特殊量化器设计 针对架构特性

  • 针对 Softmax 输出 → 自适应粒度量化 (AGQ)
    • 使用 AdaptiveGranularityQuantize + LogAvgMSEFastObserver
    • 更好地处理注意力概率分布
  • 针对 K 激活双峰分布 → 符号感知量化
    • 使用 LSQSignFakeQuantize + SignAvgMSEFastObserver
    • 检测双峰分布并调整权重符号 (BIG: Bimodal Integration)

4️⃣ 分层重建策略 工程实现

将模型分为多个重建单元（per_block / per_layer / per_stage），逐层进行重建优化：

  • Encoder 重建 (CAMS)：使用 PAR 损失，逐 stage 优化
  • Decoder 重建 (CLM)：类似策略，保护 Cross-Attention 交互
  • 重建后融合：将优化后的量化参数固定，得到最终模型

🎯 Key Takeaway - 核心要点总结

  • 范式转变： 从"数值精度优先"转向"语义对齐优先"
  • 核心创新： PCC（注意力 IoU 指导裁剪）+ PAR（跨模态交互保护）
  • 实验效果： SAM-B W6A6 量化下，比基线提升 +11.7% mAP
  • 可迁移性： 注意力 IoU 损失、交互感知重建可应用于其他 Vision Transformer 和多模态模型
  • 量化配置： 权重 per-channel + 激活 per-tensor，W4A4/W6A6/W8A8 可选
